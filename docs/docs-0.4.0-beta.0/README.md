# Blokd 0.4.0-beta.0 Documentation

Blokd is a small Hono-native framework for HTML-first applications. It provides JSX, server rendering, file-based routes, route loaders and actions, Solid-familiar signals, runtime-aware components, and two island layers:

- explicit resumable refs with `Island`, `on()`, and `defineAction()`
- compiler-assisted islands with `export const Counter = island(() => ...)`

`0.4.0-beta.0` is a public beta. It is suitable for experiments, small sites, documentation sites, form-driven apps, and Hono-backed SSR apps where small client payloads matter. APIs may still change before stable `1.0.0`.

## Install

```sh
pnpm add blokd hono
pnpm add -D vite typescript
```

Or create a starter:

```sh
pnpm create blokd@beta my-app
cd my-app
pnpm install
pnpm dev
```

## What Changed In 0.4.0-beta.0

- Compiler-assisted islands infer stable names for exported constants such as `export const Counter = island(() => ...)`.
- Route-local island client entries are generated by the Vite plugin when a route imports compiler-assisted islands.
- SSR dynamic markers are used for focused hydration boundaries.
- The client runtime can claim existing SSR DOM nodes during hydration instead of replacing the whole root.
- Static intrinsic JSX templates are compiled into cached DOM templates where safe.
- `createPages({ stream: true })` can return streamed document responses.
- Route runtime reports, route client budgets, and adapter smoke coverage are part of the hardening path.
- `Set-Cookie` headers are preserved as append-only multi-value headers when route metadata contributes headers.

## What Blokd Optimizes For

- HTML and forms first
- SSR by default
- Web `Request`, `Response`, `Headers`, `FormData`, and streams
- Hono for server composition
- small, route-scoped client JavaScript
- explicit browser behavior instead of implicit full-app hydration
- islands that can be reasoned about from route source

## What Blokd Is Not Yet

Blokd is not a mature full-stack ecosystem, a full client-side router, a React compatibility layer, or a Qwik-compatible whole-app resumability runtime. Compiler-assisted islands are intentionally focused: they are for local interactive widgets with JSON-serializable signal state and event handlers that can be replayed by rerendering the island.
